为什么需要 HMG？

AI Agent 每次重启都会忘记一切。重复犯同样的错、丢失关键上下文。HMG 让 Agent 拥有持久的、可查询的、可治理的长期记忆。

💾

存储 (Memorize)

决策、根因、约束、验证结果——以类型化图谱原子持久化，带极性、认知状态和治理标记

🔍

回忆 (Recall)

语义+关键词混合检索，图谱扩散投影，分支感知范围——在正确时间给出正确上下文

✏️

修正 (Correct)

记忆过期时完整修正——否定、确认、降级或替换，所有修正历史永久保留

🔒

治理 (Govern)

敏感知识治理——隔离、封存或提取教训。管控什么被记住，什么被遗忘

核心能力

不是简单的向量数据库或键值存储。HMG 将记忆建模为带极性、认知状态、治理和修正谱系的有类型图谱。

🧠

One-Shot Recall

一个 MCP 调用即可获取完整上下文：当前状态、先前决策、未解决风险、后续步骤。无需拼凑碎片。

🔀

分支感知范围

tenant → workspace → repository → branch 四级范围。不同分支的记忆自然隔离，合并时可追溯。

🛡️

治理控制面

隔离、封存、墓碑、提取教训。策略标签和审计上下文贯穿全链路。合规友好的记忆管理。

⚡

本地优先

Fjall 嵌入式存储，零外部依赖。数据完全在本地，无云端泄露风险。离线可用。

🔌

MCP 全工具链

8 个标准 MCP 工具：memorize、recall、correct、govern、history、handoff、agent_brief、stats。即插即用。

📐

开放协议

hmg-protocol (Apache-2.0) 独立 crate 定义记忆原子、修正、治理、范围的类型和序列化格式。

5 分钟上手

安装 → 初始化 → 第一次记忆存储 → 查询

# 1. Install
curl -fsSL https://github.com/HMG-AI/HMG-public/releases/latest/download/install.sh | sh

# 2. Initialize (writes AGENTS.md for your agent)
hmg init -g

# 3. Start daemon
hmg daemon start

# 4. Store your first memory
hmg license status

# 5. Query via MCP or HTTP API
curl -s -X POST http://127.0.0.1:3000/api/memorize \
  -H 'Content-Type: application/json' \
  -d '{"content": "My first HMG memory!"}'
# One-line install (auto-detects OS and CPU)
curl -fsSL https://github.com/HMG-AI/HMG-public/releases/latest/download/install.sh | sh

# Verify
hmg version
# hmg 0.9.2-community

# Quick health check
hmg doctor
# Add to your agent's MCP config (e.g., Codex AGENTS.md):
# {
#   "mcpServers": {
#     "hmg": {
#       "command": "hmg",
#       "args": ["--mcp"],
#       "env": {}
#     }
#   }
# }

# Or use the init command:
hmg init -g          # global install
hmg init --path .    # project-level install

💡 安装脚本会自动选择适合你系统和 CPU 的预编译二进制文件。支持 Linux (x64/ARM64) 和 macOS (Intel/Apple Silicon)。

SDK 与集成

import hmg

import { HmgClient } from "@hmg_ai/sdk-ts";
const client = new HmgClient();
await client.memorize({ content: "decision noted" });
