# Исправление и управление

HMG использует **модель исправлений и управления только с добавлением**. Атомы никогда не перезаписываются молча. Вместо этого исправления создают новые атомы с явными связями, а переходы управления сохраняют полную историю.

## Состояния жизненного цикла атомов

### Полярность

| Полярность | Значение |
|---|---|
| `positive` | Атом утверждён как истинный |
| `negative` | Атом был отрицён или заменён |
| `neutral` | Информационный — без утверждения истинности |

### Эпистемический статус

| Статус | Значение |
|---|---|
| `claimed` | Непроверенное утверждение |
| `confirmed` | Подтверждён доказательствами или авторитетом |
| `deprecated` | Более не актуален, но не ложен |
| `unknown` | Недостаточно информации для классификации |

### Состояние раскрытия (управление)

| Состояние | Извлекаемо | Значение |
|---|---|---|
| `normal` | ✅ Обычное извлечение | Состояние по умолчанию |
| `quarantined` | ❌ Скрыт от извлечения | На проверке из-за чувствительности |
| `sealed` | ❌ Скрыт, неизменяем | Ограничено юридически или политикой |
| `tombstoned` | ❌ Скрыт, нагрузка опциональна | Помечен для удаления |
| `lesson` | ✅ Только урок | Чувствительная нагрузка заменена безопасным уроком |

## Поток исправлений

Исправления создают явные рёбра `Supersedes` между атомами:

```text
Оригинальный атом (positive)
    │
    ├── negate ──→ Новый атом (negative) + ребро Supersedes
    ├── confirm_actual ──→ Полярность оригинала подтверждена + ребро Supersedes
    ├── confirm_necessary ──→ Необходимость оригинала подтверждена
    ├── demote ──→ Эпистемический статус оригинала понижен
    └── replace ──→ Новый атом (positive) + ребро Supersedes + новое содержимое
```

### Действия исправления

| Действие | Эффект |
|---|---|
| `negate` | Создаёт атом с негативной полярностью, заменяющий целевой |
| `confirm_actual` | Подтверждает фактическую точность атома |
| `confirm_necessary` | Подтверждает, что атом остаётся актуальным |
| `demote` | Понижает эпистемический статус (напр.: confirmed → deprecated) |
| `replace` | Создаёт новый атом с обновлённым содержимым, заменяет старый |


## Поток управления

Переходы управления защищают чувствительную или устаревшую память:

```text
normal → quarantined (на проверке)
quarantined → sealed (заблокирован, неизменяем)
quarantined → tombstoned (удалён из извлечения)
quarantined → normal (очищен, восстановлен)
любой → derive_lesson (заменить нагрузку безопасным резюме)
```

### Действия управления

| Действие | Из → В | Случай использования |
|---|---|---|
| `quarantine` | normal → quarantined | Подозрительно чувствительное содержимое |
| `seal` | quarantined → sealed | Юридическое удержание, комплаенс |
| `tombstone` | quarantined → tombstoned | Удалить из извлечения |
| `derive_lesson` | любой → lesson | Извлечь безопасный урок, удалить чувствительную нагрузку |


## История снимков

Каждое действие исправления и управления создаёт неизменяемый снимок, сохраняющий состояние атома на момент перехода.

## Представления извлечения

| Представление | Показывает | Случай использования |
|---|---|---|
| `normal` | Только активные атомы | Повседневное использование агента |
| `governance` | + Изолированные/запечатанные атомы | Проверка комплаенса |
| `audit` | + Все атомы включая упразднённые | Форензическое расследование |

Обычное извлечение намеренно исключает управляемые нагрузки. Аудит-извлечение показывает всё для подотчётности.
