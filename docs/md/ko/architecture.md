## 컴포넌트

### 인터페이스

HMG는 4개의 액세스 표면을 제공합니다：

| 인터페이스 | 프로토콜 | 사용 사례 |
|---|---|---|
| **MCP** | Model Context Protocol | 에이전트 통합（주요） |
| **HTTP API** | REST + JSON | SDK 통합, 커스텀 도구 |
| **CLI** | 터미널（`hmg` 명령） | 관리, 디버깅, 스크립트 |
| **TUI** | 인터랙티브 터미널（`hmg tui`） | 시각적 탐색과 관리 |

4개 표면은 동일한 기능을 노출합니다 — 기억 저장, 리콜, 수정, 거버넌스.

### 메모리 엔진

HMG의 코어. 다음을 관리합니다：

- **그래프 모델**：타입화된 엣지로 연결된 원자（Supersedes, DerivesFrom, RelatesTo 등）
- **인덱스**：키워드 검색, 시간 순서, 카테고리 그룹화, 및 시맨틱 검색
- **스토리지**：스냅샷 기록이 있는 영구 로컬 스토리지

### 에이전트 통합

`hmg init --agent <id>`로 에이전트가 HMG를 기억 레이어로 사용하도록 구성합니다. 지원되는 에이전트：

| 에이전트 | 상태 |
|---|---|
| Cursor | ✅ 지원 |
| pi (Codex fork) | ✅ 지원 |
| Claude Code | ✅ 지원 |
| Codex | ✅ 지원 |
| Windsurf | ✅ 지원 |
| Aider | ✅ 지원 |
| Continue | ✅ 지원 |

## 데이터 흐름

### 기억

```
에이전트 → "이것을 기억해：..." → HMG
  → 입력 검증
  → 타입화된 기억 원자 생성
  → 스코프와 컨텍스트 부여
  → 검색 인덱스 구축（키워드 + 시간 + 카테고리）
  → 로컬 스토리지에 영속화
  → 원자 ID + 확인 반환
```


### 리콜

```
에이전트 → "어떤 데이터베이스를 선택했지？" → HMG
  → 쿼리 의도 파싱
  → 인덱스에서 후보 검색
  → 관련성, 확실성, 최신성으로 랭킹
  → 스코프와 거버넌스 상태로 필터
  → 그래프 순회로 관련 원자 투영
  → Agent Tool Output Contract에 따라 포맷
  → 구조화된 결과 + 진단 정보 반환
```


### 수정

```
에이전트 → "그건 틀렸어, 실제로는 ..." → HMG
  → 수정 원자 생성（액션 + 이유）
  → Supersedes 엣지로 연결
  → 원본 원자의 극성/인식 상태 업데이트
  → 수정 기록 영속화
  → 수정 확인 반환
```


### 거버넌스

```
관리자 → "이 민감한 기억을 봉인해" → HMG
  → 거버넌스 액션 검증
  → 노출 상태 전환（visible → sealed）
  → 선택적으로 안전한 교훈 원자 파생
  → 거버넌스 기록 영속화
  → 원본 콘텐츠 검색 불가（sealed）
```


## 스토리지

HMG는 실행 중인 머신의 로컬에 모든 데이터를 저장합니다：

```
~/.local/share/hmg/
  stores/
    default/           ← 기본 기억 저장소
      graph/           ← 원자와 엣지 데이터
      indexes/         ← 검색 인덱스
      snapshots/       ← 수정/거버넌스 기록
```

Community 및 Developer Local 에디션에서는 데이터가 머신을 떠나지 않습니다.

## 에디션 아키텍처

HMG는 모든 에디션 코드를 포함하는 단일 바이너리입니다. 활성 에디션은 시작 시 결정됩니다：

```
HMG 바이너리
  │
  ├── 라이선스 키 없음 → Community Edition
  │     └── One-Shot Recall, 50K 원자, 5 에이전트, 수량 제한
  │
  ├── HMG_LICENSE_KEY=<your-key> → Developer Edition
  │     └── One-Shot Recall, 자동 통합, 도메인 팩, 무제한
  │
  ├── HMG_LICENSE_KEY=<your-key> → Enterprise Edition
  │     └── 모든 기능, SSO, RBAC, 멀티테넌트, 감사
  │
  └── HMG_CLOUD_TOKEN → 클라우드 연결
        └── 클라우드 인증으로 Developer 또는 Enterprise
```

즉：
- 개별 바이너리를 유지할 필요 없음
- 업그레이드는 즉시：`export HMG_LICENSE_KEY=...` 후 재시작
- Community 사용자는 Enterprise와 동일한 바이너리 품질 제공

## 보안 경계

```
┌─────────────────────────────────┐
│         에이전트 프로세스          │
│   （사용자 권한으로 실행）          │
└──────────────┬──────────────────┘
               │ MCP / HTTP (localhost)
               ▼
┌─────────────────────────────────┐
│         HMG 프로세스              │
│   （localhost:7654에 바인딩）      │
│                                  │
│   기억 데이터：사용자만 접근 가능   │
│   아웃바운드 연결 없음 (CE)        │
│   원격 측정 없음 (CE)             │
└─────────────────────────────────┘
```

- Community Edition은 **아웃바운드 네트워크 연결 제로**
- HMG는 기본적으로 `localhost`에 바인딩 — 네트워크에 노출되지 않음
- 스토리지 파일은 사용자 전용 권한 사용
