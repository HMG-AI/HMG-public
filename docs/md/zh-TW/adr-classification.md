# ADR 發布分類

根據 ADR 2026-05-24 (v2) §文件邊界，所有 ADR 在公開發布前必須進行分類。本文檔記錄每個 ADR 的分類結果。

**分類日期：** 2026-05-25

## 分類圖例

| 標記 | 含義 |
|---|---|
| **Public** | 可原樣發布 |
| **Sanitize** | 移除內部演算法/鉤子細節後可發布 |
| **Private** | 必須保留在私有 monorepo 中——涉及專有內部實作 |

## 分類

| ADR | 標題 | 分類 | 理由 |
|---|---|---|---|
| 2026-05-17 | Domain Lens 語義編譯器 | **Sanitize** | ADR v2 指出「僅發布 RFC——概念具有戰略性；編譯器細節保持私有」。移除編譯器實作，保留概念和範圍規範。 |
| 2026-05-18 | 五瓶頸擴展架構 | **Private** | 涉及儲存/索引調優內部實作和擴展架構 |
| 2026-05-18 | 召回精度實驗矩陣 | **Private** | ADR v2 指出「保持私有——涉及實驗方法論」 |
| 2026-05-19 | 記憶控制面 / 可選 LLM Gateway | **Private** | 涉及控制面架構（企業功能） |
| 2026-05-20 | Agent Brief v2 緊湊本地化 | **Sanitize** | ADR v2 指出「清洗後發布——簡報格式是協定；移除內部渲染細節」 |
| 2026-05-20 | 緊湊寫入工具確認 | **Sanitize** | 輸出格式是協定；移除內部渲染邏輯 |
| 2026-05-20 | 本地儲存路徑與守護程式 | **Public** | 公共安裝/守護程式行為——無專有內部實作 |
| 2026-05-20 | MemoryQL 治理查詢層 | **Private** | 涉及治理查詢重寫邏輯（專有） |
| 2026-05-21 | 仿生觀測整合 | **Private** | ADR v2 指出「保持私有——涉及整合架構」 |
| 2026-05-21 | 開放標準社群執行時 v1 | **Private** | 已被 v2 取代；v1 描述的是不再使用的原始碼可見模型。保留用於歷史參考。 |
| 2026-05-21 | Ratatui Hippocampus TUI | **Sanitize** | TUI 已打包在二進位中；發布概念/功能，移除內部面板渲染細節 |
| 2026-05-23 | Agent 採用合規 | **Sanitize** | ADR v2 指出「清洗後發布——採用協定是公開的；移除內部鉤子路徑細節」 |
| 2026-05-23 | Agent 工具輸出設定 | **Sanitize** | 輸出設定是 Agent Tool Output Contract（公共協定）的一部分 |
| 2026-05-24 | Agent 記憶範圍與查詢導向召回 | **Sanitize** | 範圍模型是公開的（規範 §6）；移除排名/評分內部實作 |
| 2026-05-24 | Agent 工具輸出契約 v2 | **Public** | ADR v2 指出「發布——這就是協定標準」 |
| 2026-05-24 | One-Shot Recall 引擎 | **Private** | ADR v2 指出「保持私有——涉及主要護城河」 |
| 2026-05-24 | 開放標準 v2（本 ADR） | **Private** | 包含完整商業策略、定價、防禦方案、授權草案——保持私有 |
| 2026-05-24 | v0.9.2 品質指標 | **Private** | ADR v2 指出「保持私有——涉及效能調優」 |

## 總結

| 分類 | 數量 | ADR |
|---|---|---|
| **Public** | 2 | 2026-05-20-local-store, 2026-05-24-agent-tool-output-contract-v2 |
| **Sanitize** | 6 | 2026-05-17, 2026-05-20 (brief, compact), 2026-05-21 (tui), 2026-05-23 (adoption, profiles), 2026-05-24 (scope) |
| **Private** | 11 | 2026-05-18 (兩項), 2026-05-19, 2026-05-20 (memoryql), 2026-05-21 (observation, osr-v1), 2026-05-23 (consolidation), 2026-05-24 (oneshot, osr-v2, quality) |

## 需要的操作

1. 2 個 Public ADR 可立即複製到 `export/docs/adr/`。
2. 6 個 Sanitize ADR 需要人工審查移除專有部分後才能發布。
3. 11 個 Private ADR 無限期保留在私有 monorepo 中。
